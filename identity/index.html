CoStaff Identity

One account.
Single sign-on everywhere.

A standards-based OIDC identity provider for the CoStaff platform family. It centralises who a user is — accounts, passwords, roles — and lets every platform delegate login to it over standard OpenID Connect.

01  Why it exists

Identity as a shared service.

Each platform used to carry its own users table and login — N password stores, N audit trails, N reset-password flows. Identity makes login a shared service: the identity provider owns identity, and each app owns authorization by reading roles from the token — identity and access, cleanly separated.

02  What's inside

One place to manage accounts.

Login lives in one service; every platform trusts it.

Single sign-on

Log in once, and every CoStaff platform trusts the session. Users, passwords, and realm roles live in one place.

Standards-based OIDC

OpenID Connect with authorization code + PKCE, RS256-signed tokens, and a JWKS endpoint apps use to verify them.

IdP owns identity, apps own access

The provider manages accounts and login; each app reads roles from the token and decides what each role may do.

Self-hosted admin console

A single-realm console to manage users, roles, and relying-party clients. Adding a platform is just registering another OIDC client.
