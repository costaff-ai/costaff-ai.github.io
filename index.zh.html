02  怎麼運作

你交辦之後，
AI Agent 團隊會分工合作。

在通訊軟體中，把事情交代給 Manager Agent，由它解析意圖、分派給對的專家 — 你只負責下指令、收成品。

01
你交辦
在 Telegram、LINE、WebChat 裡用一句話派工。例如「幫我看 Q3 銷售，做成 PDF」— 不用打開任何系統、不用學新介面。
02
我派工
Manager Agent 解析你的意圖，分派給對的專家組合：要看資料庫先找 DB Agent，要做報表接 BA Agent — 多個 agent 自動接力，不用手動串接。
03
專家動手
商業分析、程式、資料庫等專家 Agent 各司其職、呼叫 MCP 工具完成各自的任務 — 每個 Agent 都跑在獨立容器、可單獨擴展或替換。
04
完成
PDF、Excel、SQL 結果直接回到你的聊天視窗 — 你不需要切換工具，所有交辦與成果都在原本的對話框裡。

03  系統架構

六層分工，
每層都可獨立替換或擴展。

CoStaff 不是單一巨型 AI，而是多個 Agent + 工具串接而成的協作系統。每層都跑在自己的容器、各自負責一件事，可以獨立替換或加新組件。

04  快速開始

只需要 Docker。
用容器隔離運行，獨立又乾淨。

會跑 Docker，你就會跑 CoStaff。從 0 到能交辦工作給 AI 員工，三個指令、約 5 分鐘。

01
安裝 CLI
pip install costaff 一行裝好。前置只需要 Python 3.10+ 和 Docker，兩個都免費，多數人開發機已經有。
02
執行 costaff onboard
互動式問你幾個問題：要用哪個 AI 模型（Gemini / Claude / 自架）？要接哪個通訊軟體？貼上對應的 Bot Token 或 API 憑證。答完直接寫進 .env，不用手動編設定檔。
03
執行 costaff start
一個指令把所有容器叫起來：CoStaff Agent、MCP Server、你選的通訊軟體通道。在通訊軟體裡交辦第一個任務，AI 員工就動手做給你看 — 全程在你自己的機器上跑。

08  你的資料

跑在你的機器上，
不是我們的。

大多數 AI 工具把你的對話存在它們的伺服器，還強迫你用他們指定的模型。CoStaff 不會 — 你的資料 100% 在你的機器，要不要連 AI 推論都離線，你自己決定。

資料只在你的機器上
對話、提醒、任務、檔案，全部存在你自己機器的資料庫，任何情況下都不會同步到雲端。我們也碰不到。
AI 推論可以完全離線（可選）
用 Gemini / Claude / OpenAI：prompt 會送到該家 API，但對話歷史仍只存在你本機。
想要 100% air-gapped？接 Ollama / vLLM / LM Studio 等本機模型，連 AI 推論都不出你的網路。選擇權在你手上。
程式碼完全公開
正式上線後完整原始碼將公開在 GitHub，CoStaff 處理你資料的每個步驟，你都看得到。
