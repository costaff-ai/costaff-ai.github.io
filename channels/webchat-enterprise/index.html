CoStaff WebChat Enterprise

The browser channel,
ready for production.

The enterprise tier of CoStaff's WebChat. Same browser access as the open-source edition — file delivery and async push are shared across both — but Enterprise runs the full chatbot runtime, with multi-tenant orgs, single sign-on, and audit. Built for teams that need to run it in production.

01  The full runtime

Built on the full pipeline.

Where the OSS edition is a thin HTTP wrapper, Enterprise is a first-class adapter on the shared chatbot runtime — the same pipeline Telegram runs on — so it inherits full-duplex transport, dedup, rate-limiting, and sub-agent progress panels.

Full-duplex WebSocket

A persistent connection with reconnect and ping/pong — messages and streamed responses flow both ways in real time.

Dedup & rate-limit

Reconnects and retries never double-post, and per-user limits keep throughput production-safe.

Multi-session conversations

Each user keeps several switchable conversations — not a single thread — with chat history persisted across sessions.

File delivery & upload

Multi-file upload, plus results returned as short-lived per-user download tokens — including async delivery across machines.

02  Access & governance

Enterprise-grade access control.

Identity, multi-tenancy, and audit are built in — so one deployment can serve many teams and customers, safely.

Single sign-on

Login and roles delegated to CoStaff Identity over OIDC — JIT user provisioning, verified identity into the session, and true single-logout. TOTP 2FA supported.

Multi-tenant

Org × Human Team × CoStaff Team routing keeps customers and teams cleanly separated on shared infrastructure.

Role-based access

Roles resolved from the identity provider decide what each person can see and do — admin, member, viewer.

Audit & history

Audit logs plus persisted, queryable chat history — for compliance, review, and support.
