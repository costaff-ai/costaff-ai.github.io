CoStaff Channels

Where users talk to your staff

Channels are the front-door chat apps users hand tasks to the Manager through. Same agents, multiple chat surfaces — pick the one your team already lives in.

WebChat
Role: browser-based chat (open source)

A lightweight browser interface — no Telegram, Discord, or LINE account required. Email + password login, identity hashing built in. Drop it on your own subdomain and let your team chat with the agents straight from the browser.

  • Browser-only access — zero third-party chat app
  • Email + password login (JWT, bcrypt)
  • Identity hashed to a 16-char SHA-256 — no raw IDs stored
  • nginx + FastAPI; ships as a single Docker image
Open source View on GitHub
Telegram
Role: Telegram bot bridge

Bring CoStaff into the Telegram app your team already uses. Text, photos, and file attachments all round-trip — agent-produced PDFs, CSVs and images get auto-delivered back as Telegram attachments. New users go through an approval queue you control.

  • Text + photo + document support, both directions
  • Auto file delivery when an agent produces output
  • Identity approval workflow with rate limiting
  • /start · /reset · /help commands
Open source View on GitHub
WebChat Enterprise
Role: production-grade browser channel

Same browser entry point as WebChat — but on the full chatbot runtime so agents can deliver files, push async progress, dedup messages, and rate-limit. Multi-tenant Org / SSO / RBAC / audit log on the roadmap. For teams that need to stand this up for real customers.

  • WebSocket transport with reconnect + ping/pong
  • Agent file delivery via short-lived download tokens
  • Async-callback push, dedup, per-user rate limit
  • Roadmap: multi-tenant Org · OIDC/SAML · RBAC · audit log

Discord
Role: Discord bot bridge

A Discord bot that brings CoStaff into your server. Slash commands and plain messages both work, in both directions — and it runs on the same chatbot runtime as the Telegram channel, so behavior stays consistent across apps.

  • Discord bot — commands + messages, both directions
  • Same chatbot runtime as the Telegram channel
  • Same Manager, same agents, same identity handling
Private Contact for access
LINE
Role: LINE official-account bridge

Meet your users in the app already on their phone. Built on the LINE Messaging API webhook — hand tasks to your AI staff from the app that's already on everyone's phone.

  • LINE Messaging API webhook integration
  • A mainstream chat app across Asian markets
  • Same Manager, same agents, same identity handling
Private Contact for access
Slack
Role: Slack workspace bridge

CoStaff inside your Slack workspace. Runs over Socket Mode, so there's no public IP or inbound webhook to expose — a natural fit for self-hosted deployments behind a firewall.

  • Socket Mode — no public IP required
  • Firewall-friendly for self-hosted setups
  • Same Manager, same agents, same identity handling
Private Contact for access
Chatbot SDK
Role: shared channel runtime

The runtime SDK every channel above is built on. Identity, sessions, file delivery, and async callbacks come for free — wiring up a new chat platform takes roughly 150 lines of adapter code.

  • Shared runtime SDK behind all channels
  • Identity, sessions, file delivery, async callbacks built in
  • New platform adapter in ~150 lines
