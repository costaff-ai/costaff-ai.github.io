CoStaff Channels

使用者跟你的員工對話的地方

通訊軟體就是使用者派工給 Manager 的入口。同一批 Agent，多個聊天介面 — 選你的團隊本來就在用的那個。

WebChat
角色：瀏覽器版聊天介面（開源）

輕量瀏覽器介面 — 不用 Telegram、Discord、LINE 帳號。內建 Email + 密碼登入，身份雜湊處理好。對話關掉重開看得到，背景任務完成後結果會自動推回對話。掛到自己的子網域，團隊用瀏覽器就能直接跟 Agent 對話。

  • 純瀏覽器 — 不依賴任何第三方聊天 App
  • Email + 密碼登入（JWT、bcrypt）；身份雜湊成 16 字元 SHA-256，不存原始 ID
  • 單一 thread 持久對話 — 關掉視窗再回來，之前的問答還在
  • 非同步結果推播（SSE）—「稍後通知你」的任務會自動送回對話
  • nginx + FastAPI，單 Docker image

Telegram
角色：Telegram Bot 橋接

把 CoStaff 接進團隊本來就在用的 Telegram。文字、圖片、檔案附件雙向往返 — Agent 產出的 PDF、CSV、圖片會自動以 Telegram 附件送回。新使用者進到審核佇列由你控制。

  • 文字、圖片、文件雙向支援
  • Agent 產出檔案自動派送回對話
  • 身份審核流程 + 速率限制
  • /start · /reset · /help 指令

WebChat Enterprise
角色：商用級瀏覽器版

和 WebChat 一樣的瀏覽器入口 — 檔案派送與非同步推播兩版皆有，但 Enterprise 跑在完整的 chatbot runtime 上：全雙工 WebSocket、訊息去重、每位使用者可切換多個對話串。多租戶 Org / SSO / RBAC / 稽核 log 在 roadmap 上。要把這套搬給實際付費客戶用的團隊適用。

  • 全雙工 WebSocket 傳輸（含重連與 ping/pong）
  • 訊息去重 + 多個可切換的對話串
  • 正式環境強化：檔案派送、每用戶限流
  • Roadmap：多租戶 Org · OIDC/SAML · RBAC · 稽核日誌

Discord
角色：Discord Bot 橋接

把 CoStaff 接進你的 Discord 伺服器。指令與一般訊息都支援、雙向往返 — 而且跑在跟 Telegram channel 同一套 chatbot runtime 上，跨 App 行為一致。

  • Discord bot — 指令與訊息雙向支援
  • 與 Telegram channel 同一套 chatbot runtime
  • 同一個 Manager、同一批 Agent、同一套身份處理
Private 聯絡取得
LINE
角色：LINE 官方帳號橋接

在使用者每天都在用的 App 上服務他們。走 LINE Messaging API webhook — 用每個人手機裡都有的 LINE 直接派工給你的 AI 員工。

  • LINE Messaging API webhook 整合
  • 亞洲市場主流通訊軟體
  • 同一個 Manager、同一批 Agent、同一套身份處理
Private 聯絡取得
Slack
角色：Slack workspace 橋接

把 CoStaff 接進你的 Slack workspace。走 Socket Mode，不需要公網 IP、不用對外開 webhook — 對防火牆後的自架部署特別合適。

  • Socket Mode — 無需公網 IP
  • 防火牆後的自架環境友善
  • 同一個 Manager、同一批 Agent、同一套身份處理
Private 聯絡取得
Chatbot SDK
角色：共用 channel runtime

上面每個 channel 共用的 runtime SDK。身份、session、檔案派送、非同步 callback 都內建 — 接一個新聊天平台大約只要 150 行 adapter 程式碼。

  • 所有 channel 共用的 runtime SDK
  • 身份、session、檔案派送、非同步 callback 內建
  • 新平台 adapter 約 150 行搞定

一個 Manager，所有 channel

使用者就是使用者。
從哪個 App 傳訊息都不影響。

從 Telegram 傳的訊息與從 WebChat 傳的訊息，都會走到同一個 Manager Agent 與同一批專家團隊。
跨 channel 的使用者身份會被自動正規化 — 一個審核佇列、一份對話歷史、一批 Agent。
可同時跑任意多個 channel — costaff channel add <name>。
